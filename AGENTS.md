# BuenMouse Landing - Development Guide

## Scope

This repository contains the public Astro landing page for BuenMouse, a macOS productivity app. Keep the site static, fast, multilingual, accessible, and safe for public release.

## Security and Privacy

- Never commit secrets, tokens, private keys, internal IPs, environment dumps, local machine paths, or private infrastructure notes.
- Keep `AGENTS.md`, `README.md`, and changelog entries public-safe.
- Do not add analytics, trackers, or third-party scripts without an explicit product reason.
- Prefer strict-by-default security headers and Content Security Policy rules. If external assets are needed, add only the required domains.

## Commands

```bash

bun run typecheck
bun run build
```

Use Bun by default because the repository has a Bun lockfile. Do not commit generated `dist/` output.

## Internationalization

- `src/i18n/ui.ts` is the canonical source for copy.
- `src/i18n/utils.ts` exposes helpers such as `useTranslations()` and locale detection.
- `/en/` and `/es-ES/` are the canonical indexable routes.
- The root `/` page is a noindex redirect helper and must not be treated as a canonical landing page.
- Add or update translation keys in every supported language in the same change.

## SEO and Structured Data

- Shared metadata lives in `src/layouts/Layout.astro`.
- Keep title, description, keywords, canonical URL, `hreflang`, Open Graph, Twitter Card, theme color, CSP, and JSON-LD aligned per locale.
- The sitemap is generated by `@astrojs/sitemap` and should include only canonical indexable routes plus valid localized alternates.
- Current JSON-LD should remain consolidated and non-conflicting: `WebSite`, `SoftwareApplication`, `Person`, and `BreadcrumbList`.
- Keep `public/manifest.json` aligned with the production app name, icons, categories, `lang`, and `dir`.
- `robots.txt` should reference the sitemap and avoid blocking the canonical localized routes.

## Design System

- Global variables live in `src/styles/global.css`.
- Keep the existing dark macOS-oriented visual identity, spacing scale, and animation patterns.
- Prefer editing existing sections over adding new components unless the new component has a clear reusable responsibility.
- Respect reduced-motion behavior and avoid layout shifts in above-the-fold content.

## External Data

- `src/lib/github.ts` fetches public release information from GitHub.
- Keep network usage resilient: handle unavailable release data without breaking static generation or hydration.
- If CSP changes are required for GitHub or image/font sources, scope them to the smallest necessary domains.

## Verification

Before marking work complete, run the relevant checks:

```bash
bun run format:check
bun run typecheck
bun run build
```

For SEO work, also inspect the generated `dist/` HTML and sitemap for:

- `application/ld+json`
- canonical and alternate locale links
- `og:image`
- CSP meta tag
- valid sitemap XML

## Git Safety

- Use conventional commits.
- Ask before staging, committing, pushing, creating PRs, or merging unless the user has already granted explicit permission for the current task.
- Do not use destructive git operations unless explicitly requested.
